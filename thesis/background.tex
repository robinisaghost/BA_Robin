\section{Deep Learning for Time Series Forecasting} \label{sec:bg:models}

\subsection{Long Short-Term Memory Networks} \label{sec:bg:models:lstm}

Standard recurrent neural networks (RNNs) process sequential data by maintaining a hidden state $h_t \in \mathbb{R}^d$ that is updated at each time step from the current input $x_t$ and the previous state $h_{t-1}$, but suffer from the vanishing gradient problem: gradients decay exponentially during backpropagation through time, preventing the network from learning long-range dependencies~\citep{bengio1994vanishing}. Long Short-Term Memory networks (LSTMs)~\citep{hochreiter1997lstm} resolve this by introducing a cell state $c_t$ that is updated only additively, protected by three learnable gates: the forget gate $f_t$ controls what fraction of the previous cell state is retained, the input gate $i_t$ what new information (the candidate activation $\tilde{c}_t$) is written, and the output gate $o_t$ what part of the updated cell state is exposed as the hidden state:
\begin{align}
    f_t,\; i_t,\; o_t = \sigma\!\left(W_{\{f,i,o\}} [h_{t-1},\, x_t] + b_{\{f,i,o\}}\right)&, \qquad
    \tilde{c}_t = \tanh\!\left(W_c [h_{t-1},\, x_t] + b_c\right) \label{eq:lstm_gates} \\
    c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t&, \qquad
    h_t = o_t \odot \tanh(c_t) \label{eq:lstm_state}
\end{align}
where $\sigma(\cdot)$ is the logistic sigmoid function, $\odot$ element-wise multiplication, $[h_{t-1}, x_t]$ vector concatenation, and $W$, $b$ learnable parameters. The additively updated cell state forms a gradient highway along which information can flow over many time steps without exponential decay, making LSTMs effective for long-range temporal dependencies such as those in CGM data.

Fig.~\ref{fig:bg:lstm_arch} shows how these operations are assembled into the recurrent baseline: at each step, the current CGM reading and the previous hidden state feed the gates of Equations~\ref{eq:lstm_gates}--\ref{eq:lstm_state}; once the full 24-step lookback window is processed, a linear head maps the final hidden state to the predicted glucose value at $t + 60$~minutes. The baseline stacks two such layers; layer width and dropout are set by the hyperparameter search of Section~\ref{sec:method:setup:splits} and listed in Tab.~\ref{tab:method:hyperparams}.

\subsection{PatchTST} \label{sec:bg:models:patchtst}

Transformer-based architectures model sequence data using scaled dot-product self-attention~\citep{vaswani2017attention}, which computes pairwise dependencies between all positions in the input:
\begin{equation}
    \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
    \label{eq:attention}
\end{equation}
where $Q$, $K$, $V \in \mathbb{R}^{n \times d_k}$ are the query, key, and value matrices obtained from learned linear projections of the input tokens, and $d_k$ is the key dimension used for scaling to prevent gradient saturation of the softmax.

PatchTST~\citep{nie2023patchtst} adapts this framework for time series by replacing individual time steps with fixed-length overlapping \emph{patches} as attention tokens. Treating patches rather than scalars as tokens reduces the effective sequence length seen by the attention layers and improves the capture of local temporal patterns. Here the 24-step lookback window is divided into 4-step patches with stride 2, yielding 12 patch tokens; each token is linearly projected to an embedding and processed by a stack of transformer encoder layers, whose dimension, number of attention heads, and depth are listed in Tab.~\ref{tab:method:hyperparams}.

PatchTST applies Reversible Instance Normalisation (RevIN)~\citep{kim2022revin} to address the non-stationarity of the glucose signal: the baseline level and variability of the CGM trace drift over time, so different input windows occupy different value ranges even within a single patient's series. Before each forward pass, RevIN normalises the input window instance-wise to zero mean and unit variance:
\begin{equation}
    \hat{x}_i = \frac{x_i - \mu}{\sigma + \varepsilon}, \qquad
    \mu = \frac{1}{L}\sum_{i=1}^{L} x_i, \quad
    \sigma = \sqrt{\frac{1}{L}\sum_{i=1}^{L}(x_i - \mu)^2}
    \label{eq:revin_norm}
\end{equation}
where $L$ is the lookback length and $\varepsilon > 0$ is a small constant for numerical stability. After the model produces a normalised output, the original scale is restored by reversing the same transformation, $\hat{y}_i = \hat{y}_i^\text{norm} \cdot \sigma + \mu$. Normalising each window reduces the distribution shift between training and test windows and lets the model focus on trajectory shape rather than absolute level. This has a structural consequence for threshold-based classification: a classifier built on RevIN-normalised inputs has no access to the absolute glucose level and therefore cannot reliably associate an input pattern with the fixed 70~mg/dL threshold. The empirical manifestation is examined in Section~\ref{sec:exp:obj3} and interpreted in Chapter~\ref{ch:conclusions}.

Fig.~\ref{fig:bg:patchtst_arch} shows the complete pipeline as a bottom-to-top stack: the univariate input window is instance-normalised and patched, the patches are linearly projected to token embeddings with positional encoding and processed by the transformer encoder, and a final flatten-and-linear head maps the encoded representation back to the forecast, which the reverse step of RevIN returns to the original glucose scale.

\section{Loss Functions} \label{sec:bg:losses}
% ================================================================

This thesis evaluates several loss functions, each tied to a distinct role in the study. Mean Squared Error (Section~\ref{sec:bg:losses:mse}) is the standard regression loss and the baseline whose behaviour motivates the investigation; the offset-aware losses (Section~\ref{sec:bg:losses:offset}) are the candidate remedies for the prediction delay examined in Objective~1; and the weighted binary cross-entropy (Section~\ref{sec:bg:losses:bce}) is the classification loss used by the direct event classifiers of Objective~3.

\subsection{Mean Squared Error} \label{sec:bg:losses:mse}

Mean Squared Error is the standard objective for regression tasks and the default loss for the forecasting models in this study~\citep{goodfellow2016deeplearning}. Let $\mathbf{y} = (y_1, \ldots, y_n)$ denote the ground-truth glucose values and $\hat{\mathbf{y}} = (\hat{y}_1, \ldots, \hat{y}_n)$ the model predictions. The Mean Squared Error loss is defined as:
\begin{equation}
    \mathcal{L}_\text{MSE}(\hat{\mathbf{y}},\, \mathbf{y}) = \frac{1}{n}\sum_{t=1}^{n}\bigl(\hat{y}_t - y_t\bigr)^2
    \label{eq:mse_loss}
\end{equation}
MSE is differentiable, computationally efficient, and penalises large deviations heavily due to the squaring operation, making it a natural default for regression tasks.

In the CGM forecasting setting, however, the pointwise structure of Equation~\ref{eq:mse_loss}, which compares each prediction $\hat{y}_t$ only with the true value at the same index, has a decisive consequence: a displaced but shape-correct curve is penalised as heavily as a structurally wrong one (Fig.~\ref{fig:bg:losses}a). On the strongly autocorrelated CGM signal (Section~\ref{sec:method:setup:cgm}), a model that carries recent history forward already attains a small pointwise error~\citep{wolff2025cgpm}, so gradient descent on MSE is drawn toward this lagged solution rather than genuine anticipation. This interaction between the autocorrelated signal and the alignment-blind loss is the mathematical origin of the systematic prediction delay described in Chapter~\ref{ch:introduction}.

\subsection{Offset-Aware Losses} \label{sec:bg:losses:offset}

Objective~1 replaces MSE with a loss that tolerates small timing offsets, so that a prediction is no longer penalised merely for being a few steps early or late. Two such losses are evaluated, differing in how much temporal freedom they allow: the Bounded-Lag loss applies a fixed local tolerance, whereas Soft-DTW aligns the two sequences by warping the time axis. Both are illustrated alongside MSE in Fig.~\ref{fig:bg:losses}.

\subsubsection{Bounded-Lag Loss} \label{sec:bg:losses:bl}

The Bounded Lag loss~\citep{prg_glucose_proposal} addresses the temporal rigidity of MSE by tolerating small timing offsets: instead of comparing each predicted value $\hat{y}_t$ only against the true value at the same time step $y_t$, it selects the best-matching true value within a local window of $D$ steps in either direction:
\begin{equation}
    \mathcal{L}_\text{BL}(\hat{\mathbf{y}},\, \mathbf{y}) = \frac{1}{n}\sum_{t=1}^{n}\;\min_{|k| \leq D}\;\bigl(\hat{y}_t - y_{t+k}\bigr)^2
    \label{eq:bl_loss}
\end{equation}
A prediction correct in value but shifted by up to $D$ steps thus incurs no penalty for that offset, as the orange horizontal links in Fig.~\ref{fig:bg:losses}b show. This study sets $D = 3$ steps ($\pm$15~minutes), equal to the event-detection tolerance $\tau$ of Section~\ref{sec:bg:metrics}, so a prediction close enough to count as a correct detection at evaluation time is also treated as correct during training, weakening the MSE incentive to copy recent history. The modification is deliberately conservative, however: its fixed local window forgives small timing errors but cannot realign a prediction displaced by more than $D$ steps, which is the motivation for the second offset-aware loss.


\subsubsection{Soft-DTW: Differentiable Dynamic Time Warping} \label{sec:bg:losses:sdtw}

Dynamic Time Warping (DTW), originally developed for speech recognition~\citep{sakoe1978dtw} and later adopted for general time-series comparison~\citep{berndt1994dtw}, measures the similarity between two sequences as the cost $\langle \mathbf{A}, \Delta(\mathbf{a}, \mathbf{b}) \rangle$ of the cheapest admissible alignment $\mathbf{A} \in \mathcal{A}_{n,m}$ between them, where $\Delta_{ij} = (a_i - b_j)^2$ is the pairwise cost matrix; the time axis may be locally stretched or compressed, and the minimisation is solved by dynamic programming (see \citet{sakoe1978dtw} and \citet{berndt1994dtw} for the alignment constraints and recursion). This hard minimum makes the DTW distance unsuitable as a training loss: it is non-differentiable wherever the cheapest alignment switches paths, and where a gradient exists it reflects only the single selected alignment, so gradient-based optimisation has no stable signal to follow.

\citet{cuturi2017softdtw} resolve this with Soft-DTW, which replaces the hard minimum by a differentiable \emph{soft} minimum controlled by a temperature parameter $\gamma > 0$:
\begin{equation}
    \text{sdtw}_\gamma(\mathbf{a},\, \mathbf{b}) = \min^\gamma_{\mathbf{A} \in \mathcal{A}_{n,m}} \langle \mathbf{A},\, \Delta(\mathbf{a}, \mathbf{b}) \rangle, \qquad
    \min^\gamma(\mathbf{z}) = -\gamma \log \sum_j \exp\!\left(-\tfrac{z_j}{\gamma}\right)
    \label{eq:sdtw}
\end{equation}
The soft minimum is the log-sum-exp of the alignment costs and is smooth everywhere, so gradients flow through it. The conceptual difference from DTW is that Soft-DTW never commits to one optimal alignment: it aggregates over \emph{all} admissible alignments, weighting each by $\exp(-\langle \mathbf{A}, \Delta \rangle / \gamma)$ so that low-cost alignments dominate the sum but none is selected outright. The temperature $\gamma$ controls how sharply this weighting concentrates on the cheapest alignment: as $\gamma \to 0$ Soft-DTW recovers exact DTW, while larger values smooth the objective at the cost of discriminativeness. This study uses the intermediate value $\gamma = 1.0$.

The elastic time alignment inherited from DTW makes Soft-DTW the most direct of the three responses to the prediction delay and the central object of interest in Objective~1. Its gradient does not merely ask each predicted point to move closer in value to the true value at the same index; it asks the predicted trajectory to align with the true one in shape and in time, pushing a lagging prediction to advance toward the matching true features and thereby acting on exactly the temporal displacement that MSE leaves unpenalised. And unlike the Bounded-Lag loss, whose fixed window neutralises the penalty only within $\pm D$ steps, the warping path places no fixed bound on the offset it can absorb.

Fig.~\ref{fig:bg:losses}c shows the alignment Soft-DTW forms with a temporally displaced target, matching corresponding shape features such as peak to peak and trough to trough even when they occur at different times. This flexibility is also its principal drawback: by warping time freely, Soft-DTW can reward a prediction whose shape matches the target under warping while its step-by-step correspondence remains poor. For PatchTST the alignment is moreover computed on RevIN-normalised inputs (Section~\ref{sec:bg:models:patchtst}), in a scale-free space; the resulting effect on amplitude accuracy is visible in Section~\ref{sec:exp:obj1}.


\subsection{Weighted Binary Cross-Entropy} \label{sec:bg:losses:bce}

The three losses above are regression losses, used to train a model that predicts a continuous glucose value. The direct event classifiers of Objective~3 instead solve a binary classification problem, namely whether a hypoglycaemic step occurs within the next 60 minutes. This approach has been shown to be effective for hypoglycaemia detection by a recent end-to-end classifier~\citep{shao2024hypoglycemia} and extended to multiple prediction horizons by \citet{cinar2025hypoclassification}. They are trained with the standard loss for binary classification, binary cross-entropy~\citep{goodfellow2016deeplearning}. For a window with true label $y \in \{0, 1\}$ and predicted event probability $\hat{p} \in [0, 1]$, the cross-entropy contributes $-\log \hat{p}$ when the event occurs and $-\log(1 - \hat{p})$ when it does not, so that a confident but wrong prediction is penalised heavily while a confident correct one incurs almost no loss.

On a balanced dataset this would be sufficient. The hypoglycaemia detection task, however, is strongly imbalanced (Section~\ref{sec:method:setup:cgm}): negative windows greatly outnumber positive ones, so an unweighted loss would be dominated by the majority class and would drive the classifier toward predicting ``no event'' almost everywhere, and hence toward near-zero recall. To counteract this, the positive term is scaled by a class weight $w_{+} > 1$, giving the weighted binary cross-entropy
\begin{equation}
    \mathcal{L}_\text{wBCE} = -\frac{1}{N}\sum_{t=1}^{N} \Bigl[\, w_{+}\, y_t \log \hat{p}_t + (1 - y_t)\log(1 - \hat{p}_t) \,\Bigr].
    \label{eq:wbce}
\end{equation}
Choosing $w_{+}$ above one makes each missed positive cost proportionally more than a false alarm, which aligns the training objective with the recall-weighted F$_2$ criterion of Section~\ref{sec:bg:metrics:event}. The specific weight used for the cohort in this study, together with the way the trained classifier is applied and evaluated, is given in Section~\ref{sec:method:obj3}.

\section{Evaluation Metrics} \label{sec:bg:metrics}
% ================================================================

This study reports metrics of two complementary kinds, organised into the three parts of this section. Section~\ref{sec:bg:metrics:accuracy} covers forecasting accuracy: how close the predicted glucose values are to the truth, and how much of any error is due to timing rather than shape. Section~\ref{sec:bg:metrics:event} covers event detection: how reliably hypoglycaemia is caught. Section~\ref{sec:bg:metrics:tau} introduces the tolerance window that makes the event-detection metrics applicable to forecast-derived detectors. Together these define the columns of the results tables in Chapter~\ref{ch:experiments}.

\subsection{Forecasting Accuracy and Prediction Lag} \label{sec:bg:metrics:accuracy}

Let $g_{1:n}$ and $\hat{g}_{1:n}$ denote the ground-truth and predicted glucose sequences over $n$ test windows. The Root Mean Squared Error and Mean Absolute Error are the standard accuracy measures for glucose forecasting:
\begin{equation}
    \text{RMSE} = \sqrt{\frac{1}{n}\sum_{t=1}^{n}(g_t - \hat{g}_t)^2}, \qquad
    \text{MAE} = \frac{1}{n}\sum_{t=1}^{n}|g_t - \hat{g}_t|
    \label{eq:rmse_mae}
\end{equation}
Both are reported in mg/dL; RMSE is the more sensitive of the two to occasional large misses. In the presence of the systematic prediction delay, however, both pointwise metrics are inflated even when the predicted trajectory is structurally accurate, because it is temporally displaced~\citep{wolff2025cgpm}. This limitation motivates the lag-adjusted metric defined next.

To separate the timing component of prediction error from structural inaccuracy, a diagnostic metric~\citep{prg_glucose_proposal} is used that searches for the optimal constant temporal shift $\Delta^*$ and computes RMSE after applying it:
\begin{equation}
    \Delta^* = \operatorname*{arg\,min}_{k \,\in\, \{-K,\ldots,K\}} \text{RMSE}\bigl(g_{1:n},\; \text{shift}(\hat{g}_{1:n}, k)\bigr)
    \label{eq:best_lag}
\end{equation}
\begin{equation}
    \text{RMSE}^* = \text{RMSE}\bigl(g_{1:n},\; \text{shift}(\hat{g}_{1:n}, \Delta^*)\bigr)
    \label{eq:lag_rmse}
\end{equation}
where $\text{shift}(\hat{g}, k)$ denotes the prediction sequence shifted by $k$ steps and the search range is $K = 12$ steps ($\pm$60~minutes). RMSE$^*$ quantifies the residual prediction error once the systematic delay is corrected: the difference RMSE~$-$~RMSE$^*$ isolates the fraction of raw error attributable purely to the temporal displacement, so a low RMSE$^*$ indicates structurally accurate predictions that are merely shifted in time. The size of that shift is reported separately as the mean prediction lag $|\Delta^*| \times 5$~minutes (the \emph{Lag} column in the results tables).

\subsection{Event Detection Metrics} \label{sec:bg:metrics:event}

Hypoglycaemia detection is evaluated as a binary classification problem with the standard confusion matrix quantities: true positives (TP, a hypoglycaemia event is correctly detected), false positives (FP, an alarm is issued but no event occurs), and false negatives (FN, an event occurs but is not detected). How these counts are obtained depends on the type of detector. For the direct binary classifiers (Section~\ref{sec:method:obj3}), each window is classified independently and contributes one entry to the confusion matrix according to its predicted and true label. For the forecast-derived detectors, a predicted threshold crossing is instead matched to a true crossing within a tolerance window, as described in Section~\ref{sec:bg:metrics:tau} below. From these counts, precision $P$, recall $R$, and the $F_\beta$ score are defined as:
\begin{equation}
    P = \frac{\text{TP}}{\text{TP} + \text{FP}}, \qquad
    R = \frac{\text{TP}}{\text{TP} + \text{FN}}, \qquad
    F_\beta = (1 + \beta^2)\cdot\frac{P \cdot R}{\beta^2 \cdot P + R}
    \label{eq:f_score}
\end{equation}
Precision is the fraction of issued alarms that correspond to real events and recall the fraction of real events caught; the $F_\beta$ score combines them, with $\beta = 2$ weighting recall twice as heavily as precision. This emphasis suits hypoglycaemia detection, where a missed event is clinically more serious than a false alarm~\citep{battelino2019}; F$_2$ is therefore the primary event-detection metric reported throughout this thesis.

\subsection{Tolerance Window $\tau$ for Forecast-Derived Detection} \label{sec:bg:metrics:tau}

For forecast-derived detectors, that is, models trained as regressors whose predicted trajectory is thresholded at 70~mg/dL, requiring the predicted threshold crossing to coincide exactly with the true one is too rigid, since a prediction that is correct in shape but slightly delayed would then be scored as a complete miss. Following \citet{prg_glucose_proposal}, a tolerance window $\tau$ is therefore introduced. Writing $t_c$ for the time at which the true trajectory crosses 70~mg/dL and $p_c$ for the time at which the predicted trajectory crosses it, the detection is counted as a true positive when
\begin{equation}
    \text{TP: } |t_c - p_c| \leq \tau
    \label{eq:tau_match}
\end{equation}
Fig.~\ref{fig:bg:tau_tolerance} illustrates this rule. The standard evaluation uses $\tau = 15$~minutes (3 steps), matching the Bounded-Lag tolerance $D$ of Section~\ref{sec:bg:losses:bl}. The size of $\tau$ controls how much timing error is forgiven: at $\tau = 0$ only an exact-step crossing is accepted, so a systematically delayed forecast scores near zero, whereas at $\tau = 60$~minutes a crossing anywhere within the hour is accepted and the score reflects only whether the model produces a crossing at all. Sweeping $\tau$ across this range in Section~\ref{sec:exp:obj3} therefore separates detection failures caused by timing mismatch from those caused by a structural inability to predict the event.
